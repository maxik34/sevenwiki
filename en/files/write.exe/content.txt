<h1>write.exe</h1>
<p><img src="{{ROOT}}/icons/write-16.png"> <samp>write.exe</samp> is a legacy executable file that serves as a wrapper for the WordPad application. While it appears as a standalone program, its primary purpose is to maintain backward compatibility.</p>

<h2>Launch Behavior and Redirection</h2>
<p>The execution flow of <samp>write.exe</samp> depends on how the command is entered in the Windows interface. The system alternates between using the physical binary and the App Paths registry aliases.</p>


<h3>Direct Execution</h3>
<p>When running the binary file directly (or via Console), the program uses the <code>ShellExecuteW</code> function to call <code>wordpad.exe</code>, which launches the program specified in the registry path:</p>

<div class="pre">
	<div>Windows Registry</div>
	<pre>HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\App Paths\WORDPAD.EXE</pre>
</div>


<h3>Start Menu</h3>
<p><code>write</code> and <code>write.exe</code> launches the binary (Direct Execution), however, using <code>write.</code> launches the program specified in the registry path:</p>

<div class="pre">
	<div>Windows Registry</div>
	<pre>HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\App Paths\WRITE.EXE</pre>
</div>


<h3>Run Dialog</h3>
<p>Behaves as the inverse of the Start Menu: <code>write</code> and <code>write.exe</code> use the App Paths registry alias, while <code>write.</code> launches the binary.</p>


<h3>Explorer Address Bar</h3>
<p>All variations (<code>write</code>, <code>write.exe</code> and <code>write.</code>) launch the binary.</p>
